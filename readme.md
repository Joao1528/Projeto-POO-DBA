PROJETO Sistema de portal de frequência



2ª ENTREGA (Meta e Detalhamento)
Entrega: Sistema com as primeiras funcionalidades desenvolvidas e testadas.

Escopo da Fase Inicial: Desenvolvido o front-end de tela de login, além de validações de usuário e senhas
corretos, e validação de campo vazio e também desenvolvido o banco de dados com as tabelas e gatilhos

Atividades:
1) Desenvolver o front- end da tela de login de principal
2) Desenvolver o front-end da tela de login alunos
2) Desenvolver o front-end da tela de login professores
3) Ajustar o front-end das telas de login
4) Desenvolver back-end em php utilizando POO, para consultar as informações do banco
5) Desenvolver o back-end em php para criar estruturas de validações de login
6) Desenvolver o banco de dados com as tabelas aluno, professor, matéria e presença
7) Desenvolver usuário de banco de dados com permissão para manipular dados da tabela frequência;
8) Inserir alunos, professores e matérias nas respectivas tabelas no banco
8) Criar o usuário de banco de dados com permissão para ler dados da presença;
9) Criar gatilho no banco de dados para apagar a presença do aluno caso o aluno seja deletado
9) Montar o documento do software e colocar código no github para entrega.

3ª ENTREGA (Meta e Detalhamento)
Entrega: Sistema com todas as funcionalidades desenvolvidas e testadas.

Escopo da Fase Final: tela de login para alunos e professores contendo as validações de usuário e
senha corretos, aparentando erro caso o usuário deixe de preencher algum campo ou preencha
errado, portal para o aluno verificar suas frequências de acordo o a matéria desejada, portal do
professor para inserir a frequência dos alunos de acordo com a sua disciplina

Atividades:
1) Desenvolver o front-end do portal do aluno
2) Desenvolver o front end do portal do professor
3)Ajustar o front-end dos portais
4)Desenvolver back-end em php utilizando POO, para inserir informações do banco
5)Desenvolver back end utilizando POO, para ler informações do banco
6)Desenvolver back -end em php para editar informações do banco
7)Desenvolver back-end para atualizar informações do banco
8)Desenvolver back-end para apagar informações do banco
9)Implementar tabela virtual mostrando a frequência do aluno em matérias diferentes ou simultâneas
10) Desenvolver função que permitirá uma visualização de percentual de faltas do aluno, por parte do
professor responsável pela matéria
11) Montar o documento do software e colocar código no github para entrega.
